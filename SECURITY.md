# Security Policy

## Reporting a Vulnerability

If you believe you have found a security issue in this site or its build
pipeline, please report it privately.

- Email: `hi@mikaelkalkmann.com`
- Preferred: open a [GitHub Security Advisory](https://github.com/mikaelkkn/my-website/security/advisories/new)

Please do **not** open a public issue for security reports.

When reporting, include:

- A clear description of the issue and its impact.
- Steps to reproduce (URL, payload, expected vs. observed behavior).
- Any relevant logs, screenshots, or PoC.

I aim to acknowledge reports within 7 days and to remediate confirmed issues
as quickly as is reasonable given that this is a personal project.

## Scope

In scope:

- The published site at `https://mikaelkalkmann.com`.
- The source in this repository (build, workflows, configuration).

Out of scope:

- Vulnerabilities in upstream platforms (GitHub Pages, Cloudflare, browser
  vendors). Please report those to the respective vendor.
- Reports based solely on missing best-practice headers without a
  demonstrated impact.
